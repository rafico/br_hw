BR Summary V2

The upgraded pipeline keeps the original end-to-end structure but replaces the weakest baseline links: BoT-SORT adds appearance-aware tracking, ReID supports an OSNet plus CLIP ensemble, stage-1 clustering adds reranking and a co-occurrence rule, stage-2 clustering switches to top-K keyframes plus k-medoids and HDBSCAN, and scene classification can use Gemini with grounded JSON events and person IDs.

Approach

  • Detection and tracking: YOLO11m plus BoT-SORT with shared embeddings.
  • ReID: OSNet, CLIP, or concatenated ensemble features with normalized caching.
  • Clustering: legacy union-find path is preserved; V2 adds multi-prototype distances, re-ranking, HDBSCAN, and torso-color tie-break reassignment.
  • Scene labeling: VideoMAE remains available, while Gemini can emit strict JSON crime events, timestamps, rationale, and involved global IDs.
  • Evaluation: frame-level clustering metrics plus clip-level scene metrics and crime-span IoU.

Ablation Template

StageExpected Change

Limitations

  • Handheld camera motion still hurts both tracking and scene grounding.
  • 768x432 video resolution limits part-level and texture-heavy cues.
  • Gemini requires network access and a valid API key; the offline fallback is only partially wired.

Next Steps

  • Add cross-camera spatio-temporal priors for better re-entry reasoning.
  • Use segmentation-aware crops for more occlusion-robust embeddings.
  • Fuse audio if a future version includes it.
